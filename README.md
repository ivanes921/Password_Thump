# Password Thump

Небольшое SwiftUI‑приложение под iOS, которое имитирует ввод экранного пароля с выбором длины кода (4 или 6 цифр), загрузкой обоев и особой логикой реакции на попытки:

- При выборе 4‑ или 6‑значного кода открывается экран набора пароля, стилизованный под экран блокировки iPhone.
- Первые две попытки ввода любого кода завершаются анимацией ошибки.
- На третьей попытке приложение выставляет бейдж на иконке (число = введённый код − 2025, не ниже 0) и завершает работу, возвращая пользователя на домашний экран.
- Можно выбрать фото из медиатеки, чтобы использовать его как фон экрана блокировки.

## Структура
- `App/PasswordThump/PasswordThumpApp.swift` — точка входа приложения и управление разрешением на бейджи.
- `App/PasswordThump/ContentView.swift` — выбор длины пароля и загрузка изображения через `PhotosPicker`.
- `App/PasswordThump/LockScreenView.swift` — экран ввода кода с «точками», цифровой клавиатурой, ошибками и вычислением бейджа.

## Как собрать в Xcode
1. Создайте новый проект **iOS App** c интерфейсом **SwiftUI** и названием `PasswordThump`.
2. Замените сгенерированные `PasswordThumpApp.swift`, `ContentView.swift` и добавьте `LockScreenView.swift` из папки `App/PasswordThump/`.
3. В **Signing & Capabilities** убедитесь, что у приложения есть разрешение на **Badges** (запрос выполняется в коде через `UNUserNotificationCenter`).
4. Соберите и запустите на iPhone/симуляторе под iOS 17+ (для `PhotosPicker`).

> **Важно:** приложение намеренно завершает работу на третьей попытке ( `exit(0)` ), чтобы вернуть пользователя на домашний экран, где будет виден бейдж с рассчитанным числом.
